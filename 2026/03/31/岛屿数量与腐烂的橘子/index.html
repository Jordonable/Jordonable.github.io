岛屿数量与腐烂的橘子

Created2026-03-31|Updated2026-03-31|算法题解
|Post Views:

前言

这两道题是图遍历算法的经典代表,分别展示了 DFS 和 BFS 的典型应用场景。第一次做确实需要时间理解,但掌握后就是标准模板了。

母题模型识别

凡是下面这些题,第一反应都可以是 DFS/BFS:

  • 岛屿数量
  • 最大岛屿面积
  • 被围绕的区域
  • 图像渲染
  • 单词搜索
  • 连通块统计

共同点: 从一个起点出发,把一整片相连区域全部扩散访问掉。

题解

思路解析

核心思想

这道题的本质是连通块计数。每发现一块陆地,就用 DFS 把整个岛屿”标记”掉,避免重复计数。

算法流程

  1. 外层双重循环: 遍历整个网格,相当于”直升机扫描全图”
  2. 发现陆地: 遇到 '1' 时,说明发现了新岛屿,计数器 +1
  3. DFS 标记: 立即用 DFS 把这个岛屿的所有相连陆地都变成 '0',防止后续重复计数
  4. 递归终止: 遇到边界或水域时停止扩散

易错点总结

1. 边界条件判断顺序

2. 忘记标记已访问

如果不把访问过的 '1' 改成 '0',会导致:

  • 同一个岛屿被重复计数
  • DFS 陷入无限递归(栈溢出)

3. 计数时机错误

4. 方向数组遗漏

DFS 必须向四个方向扩散(上下左右),少写一个方向会导致岛屿标记不完整。

题解 2: 腐烂的橘子 (BFS)

思路解析

核心思想

这道题是多源 BFS 最短路径问题。关键在于:

  • 所有腐烂橘子同时开始传播
  • 需要计算最少分钟数,而不是统计数量
  • 必须按”层”扩散,每一层代表一分钟

算法流程

  1. 初始化: 遍历网格,把所有腐烂橘子加入队列,统计新鲜橘子数量
  2. 边界情况: 如果一开始就没有新鲜橘子,直接返回 0
  3. BFS 扩散: 每次处理队列中的一整层(当前分钟的所有腐烂橘子)
  4. 感染邻居: 向四个方向扩散,把新鲜橘子变腐烂,加入队列
  5. 时间推进: 处理完一层后,分钟数 +1
  6. 结果判断: 如果还有新鲜橘子剩余,返回 -1,否则返回分钟数

易错点总结

1. 没有锁定当前层的大小

这是最容易出错的地方! 如果不锁定 size,新感染的橘子会在同一分钟被处理,导致时间计算错误。

2. 时间推进的位置错误

3. 忘记检查 freshCount

4. 初始状态判断遗漏

如果一开始就没有新鲜橘子,应该返回 0 而不是进入 BFS 循环:

5. 标记已访问的方式

必须在加入队列前就把橘子标记为腐烂(grid[r][c] = 2),否则同一个橘子可能被重复加入队列。

DFS vs BFS 选择指南

DFS 适合

  • 统计连通块数量(岛屿数量)
  • 计算区域面积
  • 路径是否存在
  • 全局染色/标记
  • 一条路走到底的场景

特点: 递归实现简洁,空间复杂度 O(递归深度)

BFS 适合

  • 最短路径/最少步数
  • 层级扩散(腐烂的橘子)
  • 多源同时传播
  • 需要按”距离”分层处理

特点: 队列实现,空间复杂度 O(当前层节点数)

总结

这两道题虽然都是图遍历,但解题思路完全不同:

  • 岛屿数量: 用 DFS 深度优先,一次性标记整个连通块
  • 腐烂的橘子: 用 BFS 广度优先,按时间层级扩散
