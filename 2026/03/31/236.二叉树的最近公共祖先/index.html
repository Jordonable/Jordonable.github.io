236.二叉树的最近公共祖先

Created2026-03-31|Updated2026-03-31
|Post Views:

题解

思路

  • 利用递归回溯时，信息只能从下往上汇聚来找信息交汇处.
  • 思路是p,q失踪了,全员寻找,如果有该节点的孩子回复不是null,此节点就说我的左/右孩子知道他的信息,如果他父亲开始问了,就报告给他的父亲.层层上报直到有一个节点知道它左右孩子都有信息.
  • 这题里“后序”不是为了遍历顺序，而是为了“决策时机”
  1. 左边先汇报
  2. 右边再汇报
  3. 当前节点根据两份汇报做决定
    这就是后序。

真难吧,想不到.

1. 什么时候用前序

当前节点要先做事，再把影响传给孩子。

关键词：

  • 先访问自己
  • 父亲给孩子传信息
  • 路径、深度、前缀和、状态下传
  • 建树、复制树、序列化

你可以记成：

前序 = 自上而下

典型题：

  • 路径总和
  • 前缀和 + 回溯
  • 二叉树展开为链表的栈版
  • 记录从根到当前路径

一句判断法：

如果孩子要用到我现在的信息，就前序。

2. 什么时候用中序

只有当“左-中-右”这个顺序本身有意义时。

最常见就是 BST。

因为 BST 的中序遍历结果是：

从小到大

你可以记成：

中序 = 排序 / BST 专用感最强

典型题：

  • 验证二叉搜索树
  • 二叉搜索树第 k 小
  • 把 BST 转成有序数组

一句判断法：

如果题目和 BST 的有序性有关，优先想中序。

补一句很重要的话：

普通二叉树里，中序通常没前序和后序常用。

3. 什么时候用后序

我要先拿到左右子树的结果，再决定当前节点怎么办。

关键词：

  • 先问孩子
  • 再汇总自己
  • 高度、深度、平衡
  • 最近公共祖先
  • 子树信息汇总

你可以记成：

后序 = 自下而上

典型题：

  • 二叉树最大深度
  • 平衡二叉树
  • 最近公共祖先
  • 直径
  • 路径和的一些“子树汇总”题

一句判断法：

如果当前节点的答案依赖左右孩子的答案，就后序。

一眼判断口诀

你就背这个：

  • 信息往下传：前序
  • 利用 BST 有序性：中序
  • 信息往上收：后序

你最近做过的题，对应起来就是

  • Path Sum III 前缀和：前序进入节点记账，后序离开节点回溯擦掉
  • Lowest Common Ancestor：后序，因为必须先知道左右子树找到了什么
  • Flatten Binary Tree：
    • 用数组存前序顺序再重连：本质是前序
    • 用“返回尾巴”原地拼接：本质是后序

最实用的判断方式

做树题先问 2 个问题：

  1. 当前节点要不要立刻处理？
    要，就想前序。

  2. 我是不是必须先知道左右子树的结果？
    是，就想后序。

如果题目又提到 BST、升序、前驱后继、k 小，优先想中序。
